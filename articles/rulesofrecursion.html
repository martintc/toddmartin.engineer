Rules of Recursion

In another blog article, I touched briefly on what recursion looks like and how it flows. I touched on how it acts as a loop with a little but more involvement when it comes to using control statements. Here, I will touch on some of the rules and some other key terms of recursion.

Recursion must stop

Recursion must always come to an end. If not, it will result in an infinite loop. This of writting a loop like the one below:


while (true) {

	System.out.println("Hello!");

}

The above code will print out “Hello!” for ever until the program is interrupted by some outside force. It will not stop on it’s own.

Luckily, there are some programming languages that will check for this conditions and spit out an error. Java is one, however, it isn’t exactly pretty.


public String nonStopRecursion () {

	System.out.println("Hello!");

	return nonStopRecursion();

}

This method above will lead to infinite recursion. Java will compile it, and allow it to run, but it will eventually stop running and begin to spit out quiet a few errors, then it will eventually top that. Below is a snippet of my output, not the full output, it was much longer.

In order to define a stopping point for recursion, a base case must be defined. This is covered in the section below.

Base case

The base case is fundamental to writing recursive functions. It is essentially the stopping point for recursion. We need to think of recursion as a chain of events. The base case is the stopping point at which it will return up the chain.

Look at example 5 in my previous article: A quick glace at recursion

In the implementation of the recursive method factorial, the base case is defined by the line:


if (n == 1) {



    return 1;



}

At this point, the method will not longer return itself with a different set of parameters. Instead it will return a definite value of 1. Recursion has stopped.

Problem that is broken down

Recursion is typically used as a way to solve bigger problems, but to can be broken down into chunks. It also holds the advantage of creating a solution to a problem that is more legible. When thinking about breaking down a problem, think about how the base case is the point at which a problem can no longer be broken down anymore. Each time a problem is broken down into a smaller problem, the smaller problem’s solution must contribute to the solution of the larger problem. Finally, the solution to the original problem must result from the solutions of all smaller problems.

To obtain the factorial of 5, we break the problem all the way down to 1. Then we multiply that by 2. The product of that is multiple by 3, and so forth as is shown below.


Factorial of 5

5 – 1 = 4

4 – 1 = 3

3 – 1 = 2

2 – 1 = 1

1 x 2 = 2

2 x 3 = 6

6 x 4 = 24

24 x 5 = 120
